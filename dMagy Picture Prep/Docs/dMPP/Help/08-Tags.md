# Tags

Tags are reusable labels you can apply to pictures.

Use tags for categories that help you organize, filter, review, or prepare pictures later.

## Why Tags matter

Tags help describe what a picture is about without writing a long description every time.

They can help with:

- finding related pictures
- marking pictures for follow-up
- filtering pictures for future tools
- excluding pictures from display
- grouping pictures by event, theme, or use

## What makes a good tag

Good tags are reusable.

Use tags for categories that could apply to many pictures.

Examples:

- Christmas
- Halloween
- Wedding
- School
- Military
- Vacation
- Do Not Display
- Flagged

Avoid creating tags that are really titles or descriptions.

Better as a tag:

```text
Christmas
```

Better as a description:

```text
Opening presents at Grandma Jean’s house, probably Christmas Eve.
```

## Tags vs. Description

Use Tags for broad categories.

Use Description for the story of this specific picture.

Example:

```text
Tags:
Christmas, Family

Description:
Anna, Zach, and Hannah opening presents in the living room at Grandma Jean’s house.
```

The tags help you find or filter the picture later. The description preserves the specific memory.

## Tags vs. Curator Notes

Use Tags for searchable categories.

Use Curator Notes for curator-facing notes, uncertainty, repair clues, or follow-up tasks.

Example tag:

```text
Flagged
```

Example Private Note:

```text
Ask Mom whether this was Christmas 1997 or 1998.
```

## Built-in and reserved tags

dMPP includes built-in or reserved tags that are intended to work consistently across dMagy apps.

These tags are part of the shared dMagy archive language. They help dMPP and future dMagy tools understand common review and display decisions in the same way.

Reserved tags have special meaning in the app.

For example:

```text
Flagged
```

may be used as a review option for pictures that need follow-up.

Another reserved tag is:

```text
Do Not Display
```

which can help future display tools avoid showing pictures that should be kept out of slideshows or public views.

Reserved tags may not be removable or renameable, though their descriptions may still be editable.

## Tag descriptions

Tags can include descriptions.

A description explains how the tag should be used.

Example:

```text
Tag:
Flagged

Description:
Use for pictures that need follow-up, correction, identification, or later review.
```

Descriptions are especially helpful when a tag could be interpreted more than one way.

## When to add a new tag

Add a new tag when:

- you expect to use it on many pictures
- it helps with filtering or review
- it describes a recurring event, theme, or use
- it helps future display or slideshow decisions

Do not add a new tag for every small detail.

If the detail is unique to one picture, use the Description or Curator Notes field instead.

## Use tags sparingly at first

It is easier to add more tags later than to clean up many overlapping tags.

For example, you probably do not want all of these unless you have a clear reason:

```text
Christmas
Christmas Eve
Christmas Morning
Xmas
Holiday
Holidays
Winter Holiday
```

Start with a few broad, useful tags. Refine later as patterns emerge.

## Unknown or missing tags

A picture sidecar may contain a tag that is no longer defined in Settings > Tags.

When this happens, dMPP may show a warning that the picture includes tags not currently in Settings.

You can usually choose to:

- add the tag back into Settings
- remove the tag from the current picture

This helps keep picture metadata and the shared Tags list aligned.

## Tags in Settings

Use **Settings > Tags** to manage the tags available in the editor.

In Tags Settings, you can:

- add tags
- edit tag names
- add descriptions
- remove tags you no longer need
- review linked tag data when needed

The Tags list is shared across the whole Picture Library Folder.

## Tags are archive-wide

Tags are stored in the portable archive data for your Picture Library Folder.

That means the same Tags list is available across all working folders inside that archive.

For example, a `Christmas` tag can be used in:

```text
Family Photos / 1970s
Family Photos / 1980s
Family Photos / Scanned Albums
Family Photos / Phone Photos
```

## Practical examples

### Event tag

```text
Tag:
Wedding
```

Use this for pictures from weddings or wedding-related events.

### Review tag

```text
Tag:
Flagged
```

Use this for pictures that need follow-up.

Examples:

- uncertain date
- unknown person
- location needs checking
- crop needs improvement
- possible duplicate

### Display-control tag

```text
Tag:
Do Not Display
```

Use this for pictures that should not appear in slideshows or public family displays.

The picture can remain in the archive while future display tools avoid showing it.

## Good habits

Use tags consistently.

Prefer one clear tag:

```text
Christmas
```

over several near-duplicates:

```text
Christmas
Xmas
Holiday Pics
Christmas photos
```

Use descriptions to clarify how tags should be used.
