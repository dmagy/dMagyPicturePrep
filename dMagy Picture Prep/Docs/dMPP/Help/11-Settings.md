# Settings

Settings manage the shared building blocks dMPP uses across your Picture Library Folder.

These settings are not just preferences for one picture. They help define the reusable People, Locations, Tags, and Crops available throughout the archive.

## Why Settings matter

Settings help keep your archive consistent.

Use Settings to manage:

- People
- Locations
- Tags
- Crops
- General archive information
- Linked files and advanced storage details

Most users do not need to adjust Settings constantly. Start simple, then return to Settings as your archive grows.

## General

Use **Settings > General** to review basic archive information.

This may include:

- the current Picture Library Folder
- the location of `dMagy Portable Archive Data`
- tools for showing folders in Finder
- tools for copying file names or paths
- startup or Getting Started options

General Settings are useful when you want to confirm where dMPP is storing archive data.

## People

Use **Settings > People** to manage the people used across your archive.

You can add and edit people, including information such as:

- names
- preferred names
- aliases
- birth dates
- death dates
- notes
- family relationships
- identity versions over time

You do not need to fill in everything for every person. Add what helps you review and understand pictures.

People settings are especially useful for:

- age clues
- name-at-time-of-picture behavior
- Suggested mode
- consistent people tagging

## Identity versions

A person can have different names or identities over time.

For example:

```text
Before 1996:
Amy Johnson

After 1996:
Amy Miller
```

Identity versions help dMPP show the name that best fits the date of the picture while still treating those names as the same person.

Use identity versions when a name change or life-stage label will make the archive clearer.

## Suggested mode default

People Settings include a default People mode for new, never-reviewed pictures.

The default can be:

- Manual
- Suggested

Use Suggested when you usually want dMPP to look for faces first.

Use Manual when you prefer to identify people yourself or when you often work with group photos, old scans, or pictures where face detection is less reliable.

You can still switch modes while reviewing a picture.

## Learned face samples

As you confirm people in Suggested mode, dMPP can learn from saved assignments.

People Settings may include tools to clear learned face samples for a person when suggestions become incorrect.

Use this carefully. Clearing learned samples removes dMPP’s learned face examples for that person, but it does not delete the person from your archive.

## Locations

Use **Settings > Locations** to manage saved places.

A saved Location can include:

- Short Name
- Description
- street address
- city
- state
- country

Create saved Locations for places you expect to reuse.

Examples:

- Home
- Grandma’s House
- Church
- School
- Lake Cabin
- New Ulm Cemetery

Saved Locations make location names consistent and help turn GPS-derived addresses into more meaningful places.

## Tags

Use **Settings > Tags** to manage the tags available in the editor.

You can:

- add custom tags
- edit tag names
- add tag descriptions
- remove tags you no longer need
- review or repair tags when needed

dMPP includes built-in or reserved tags that are intended to work consistently across dMagy apps.

Reserved tags may have standard meanings for review, filtering, or display. Because of that, they may not be removable or renameable, though their descriptions may still be editable.

## Crops

Use **Settings > Crops** to manage crop presets and default crop behavior.

You can use Crops Settings to control which crops are created for new pictures.

Examples:

- Landscape 16:9
- Portrait 4:5
- Square 1:1
- custom crop presets

Default crops give each new picture a useful starting point.

Custom crop presets are useful when you often need a specific shape for printing, display, or a project.

## Linked files

Some Settings sections may include a **Linked file** or **Linked file (advanced)** area.

This shows where dMPP stores that part of the portable archive data.

Examples may include:

- People registry
- Locations registry
- Tags registry
- Crops registry
- Face index data

Most users do not need to open these files.

Use linked file information when:

- you want to confirm where data is stored
- you are backing up or inspecting the archive
- you are troubleshooting
- someone helping you asks for the file path

These files are part of the portable archive model. They live inside your Picture Library Folder under `dMagy Portable Archive Data`.

## Shared archives and Settings

If your Picture Library Folder is stored in a shared folder service such as Dropbox, more than one person may be able to use the same archive.

dMPP can help reduce collisions by warning when another user appears to be working with the same picture or shared Settings area.

This is helpful, but it is not the same as real-time collaboration.

Good shared-archive habits:

- avoid editing the same Settings section at the same time as another user
- let Dropbox or your sync service finish syncing before switching users
- pause if dMPP warns that another user may be editing something
- coordinate before making large changes to People, Locations, Tags, or Crops

## Changing or refreshing the Picture Library Folder

Use **Change or Refresh Picture Library Folder…** when:

- you need to change the archive root
- you moved the archive
- macOS or cloud storage needs you to refresh folder access
- dMPP cannot read or write portable archive data

Refreshing access means selecting the same Picture Library Folder again.

This can restore macOS permission without changing your archive.

## Good habits

Start with the minimum Settings you need:

1. Add key People.
2. Add saved Locations.
3. Review pictures.
4. Return to Settings as patterns emerge.

Do not try to perfect every Setting before you begin. dMPP is designed to let your archive become more useful over time.
