# Locations

Locations in dMPP are shared across your whole Picture Library Folder.

A saved Location gives you a consistent, human-friendly way to describe where pictures were taken or what place they show.

## Why Locations matter

Locations help preserve the meaning of a picture.

A folder or file name might tell you something general, but a saved Location can make the place clear and reusable.

Locations can help with:

- finding pictures from the same place
- keeping place names consistent
- understanding family history
- improving GPS-derived address information
- preparing future slideshows, displays, or searches

## Saved Locations

Create a saved Location when you expect to use the place more than once.

Good examples include:

- Home
- Grandma’s House
- Christ Our Savior
- Lake Cabin
- New Ulm Cemetery
- Longmont House
- Elementary School

A saved Location can include:

- Short Name
- Description
- Street address
- City
- State
- Country

You do not need to fill in every field. Start with the information that will help you recognize and reuse the place.

## Short Name

The Short Name is the human-friendly name for the place.

Examples:

```text
Grandma’s House
Church
Lake Cabin
New Ulm Cemetery
```

Use the name people would naturally understand.

## Description

The Description gives extra context.

Examples:

```text
Jean and Robert’s house on Maple Street.
Where the family gathered for Christmas in the 1980s.
Christ Our Savior Lutheran Church in Longmont.
```

Descriptions are helpful when a short name alone is not enough.

## Structured address fields

The address fields give dMPP more structured place information.

These can include:

- street address
- city
- state
- country

Use these fields when you know them, especially for places you expect to reuse.

Structured address information can make saved Locations easier to match, compare, and understand later.

## Saved Location vs. manual location

A saved Location is reusable across the archive.

Manual location details are useful when the place is specific to one picture and you do not expect to use it again.

Use a saved Location for places like:

- Grandma’s House
- Church
- School
- Family Cabin

Use manual details for one-time places like:

- roadside scenic stop
- unknown park
- hotel from one trip
- temporary event location

## GPS-derived locations

Some pictures contain GPS information from a phone or camera.

When GPS information is available, dMPP can use it to help fill in location details.

GPS can be useful, but it is not always perfect.

A picture taken at one house may resolve to a nearby address. A picture taken in a park may resolve to a road, building, or nearby landmark.

Use GPS as a helpful starting point, not as final proof.

## Reset to GPS

Use **Reset to GPS** when you want dMPP to rebuild the location from the picture’s GPS information.

This can be useful when:

- the location was cleared
- the location was edited manually
- you want to compare GPS data with a saved Location
- you want to see what address dMPP can derive from the photo

If dMPP finds a saved Location that appears to match the GPS-derived address, it may use the saved Location to make the result more meaningful.

## Saved-location matching

Saved Locations help turn raw address data into meaningful places.

For example, GPS might return a street address, but your saved Location may say:

```text
Grandma’s House
```

That is usually more useful than the raw address alone.

dMPP may also match a saved Location when the GPS-derived address is slightly different but appears to refer to the same place, such as a nearby house number on the same street.

This helps with common GPS and reverse-geocoding drift.

## When GPS seems slightly wrong

GPS and reverse geocoding are not exact.

If the address is close but not quite right:

1. Check whether you already have a saved Location for the real place.
2. Select the saved Location if it matches the picture.
3. Edit the address fields if needed.
4. Use the Description field to clarify the place.

The saved Location should represent the meaningful place, not just the raw GPS result.

## When to create a saved Location

Create a saved Location when:

- the place appears in many pictures
- the place has family or historical meaning
- you want consistent naming
- GPS often resolves it awkwardly
- you want to search or filter by that place later

Do not create a saved Location for every single one-time place unless it will help you later.

## Good habits

Use names you will recognize later.

Prefer:

```text
Grandma Jean’s House
```

over:

```text
House
```

Prefer:

```text
New Ulm Cemetery
```

over:

```text
Cemetery
```

A good saved Location should be specific enough to be useful but broad enough to reuse.

## Troubleshooting

If Locations seem missing, dMPP may not be using the Picture Library Folder you expected, or macOS may not be allowing access to the portable archive data.

Try **Change or Refresh Picture Library Folder…** and select the same Picture Library Folder again.

If GPS gives a nearby but incorrect address, select the saved Location that represents the real place or edit the location manually.
